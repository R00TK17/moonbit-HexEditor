项目简介
MoonBit Hex Editor 是基于 MoonBit 语言从零开发的跨平台终端十六进制编辑器与二进制
文件分析工具，面向二进制调试、文件解析、安全取证、自动化运维等场景。项目同时搭载
交互式 TUI 界面与完整 CLI 命令体系，依托 MoonBit 多后端编译特性，一套代码可分别
编译为桌面原生终端程序、Web Wasm 精简程序，适配多端使用需求。项目通过 C FFI 完
成 Windows、Linux 全平台终端底层适配，保障终端交互稳定性与兼容性，可实现十六进制
查看、编辑、检索、多格式文件解析、隐写数据检测与分离等能力，同时配套完善的测试用
例，既是实用的二进制工具，也是展示 MoonBit 在系统编程、跨平台 TUI、FFI 领域能力
的生态示范项目。 核心功能范围
实现标准交互式 TUI 界面，采用经典十六进制展示布局，支持方向键导航、翻页、地址跳
转、内容高亮、光标定位；支持十六进制 / ASCII 双模式输入，提供单字节修改、插入、删
除、撤销、保存等全套编辑操作，支持搜索定位与文件结构视图切换。
配套完整 CLI 命令集，包含文件查看、文件信息检测、模式检索、文件结构解析等子命令，
支持自定义参数配置，可无缝接入脚本、管道等自动化工作流。
自研动态字节容器 HexBuffer，支持二进制数据读写、增删、替换、填充等基础操作，实现
灵活的格式化输出、正向 / 反向字节匹配检索算法。
内置解析器，自动识别并解析图像、音频、视频、压缩包、可执行文件共计 16 种主流格式，
提取文件签名、编码参数、内部结构、加密状态等核心信息。
参考 binwalk 实现文件隐写检测功能，可识别文件拼接、尾部附加数据等隐写场景，支持
隐藏数据一键分离与提取。
完成跨平台终端 I/O 适配，Windows 基于控制台 API + ANSI 转义码开发，Linux 基于
termios 原始模式实现，配置信号处理器，异常退出时自动恢复终端原始状态。
编写 39 个回归测试用例，全面覆盖字节操作、格式化输出、检索、交互式编辑等核心模块，
保障程序运行稳定。
提供完整项目文档、使用示例，说明编译方式、TUI 快捷键、CLI 命令用法与各平台适配说
明。
移植或参考说明
本项目为完全原创项目，未移植、复刻任何现有项目代码。 参考项目名称：hexyl、HxD、
xxd、binwalk 参考依据：借鉴主流十六进制编辑器、二进制分析工具的交互逻辑、功能设计
思路；文件解析遵循 PNG、ZIP、ELF 等格式官方公开标准；终端底层开发参考 Unix termios
手册、Windows Console API 官方文档。
本项目许可证：Apache-2.0
与参考工具相比，本项目结合 MoonBit 语言特性做定制化设计与改造：
1. 全程使用 MoonBit 原生语法、包结构、模块划分组织代码，构建轻量化、模块化的工程
架构；
2. 依托 MoonBit 多后端编译能力，区分原生端与 Wasm/JS 后端，原生端提供完整 TUI 交
互，Wasm 端保留精简 CLI 能力，实现一套代码多端部署；
3. 通过 C FFI 对接系统底层终端接口，针对性优化跨平台兼容性与信号安全处理；
4. 弱化图形界面依赖，聚焦终端场景，同时兼顾交互式操作与自动化命令行两种使用模式；
5. 内置完整回归测试体系，贴合 MoonBit 生态的开发与运维规范。
